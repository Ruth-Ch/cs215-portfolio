**Week 11 update**
Data Source Selected

For my final project, I decided to study how people evaluate responses from large language models (LLMs) and how prompt design affects response quality. Specifically, I will compare responses generated by ChatGPT and Gemini across different prompt categories and analyze how human reviewers score them.

Rather than using a pre-existing dataset, I will create my own dataset because my research question is specific to response quality and prompt design. My dataset will include:

Four prompts across categories such as factual, coding, analytical, and creative prompts
Responses generated by two LLMs (ChatGPT and Gemini) for each prompt
Human evaluation scores from reviewers using a scoring rubric

I chose this project because LLMs are becoming widely used in education and professional work, but response quality varies depending on the model and the prompt structure. I want to better understand how people judge these differences.

Data Acquisition Plan

Since this dataset does not already exist, I will create it manually.

First, I will finalize four prompts across different categories. Then I will submit the same prompts to both ChatGPT and Gemini and record their responses.

I will organize this information in a spreadsheet with columns such as:

Prompt category
Prompt text
Model used
Response text
Response length
Response generation time

Next, I will ask a small group of student reviewers to score each response using a simple rubric based on:

Accuracy
Clarity
Completeness
Helpfulness
Overall quality

This will allow me to compare both model performance and reviewer consistency.

Preliminary Datasheet Audit
Motivation

This dataset is being created to answer the question: how does prompt design affect perceived LLM response quality across models?

It is designed specifically for this project because there is no existing dataset that directly compares these prompt categories, model responses, and human evaluation scores together.

This project is self-directed for my Data Science final project and is not externally funded.

Composition

The dataset will be a small sample rather than a complete representation of all possible prompts or model outputs.

It will include only four prompts and two models, so it will not represent all use cases of LLMs.

Possible sources of noise include reviewer subjectivity, differences in interpretation, and variation in how people define “good” responses.

Prompt categories may also influence scoring differently.

Collection Process

I will personally collect the prompt responses by entering the same prompts into each model and recording the outputs.

Student reviewers will voluntarily score the responses using a rubric.

Because reviewers are providing feedback rather than personal sensitive data, formal consent is informal but participants will understand the purpose of the project before contributing.

Uses and Limitations

This dataset is useful for comparing perceived response quality across models and prompt types.

It should not be used to make broad claims about all LLM performance or all users because the sample size is small and the evaluation is subjective.

It is better suited for exploratory analysis rather than universal conclusions.

Preliminary Exploratory Analysis

So far, I have started testing prompt categories and reviewing sample responses.

I noticed that factual prompts are easier to evaluate because correctness is clearer, while creative prompts are more subjective and harder to score consistently.

Coding prompts often produce technically correct answers from both models, but explanation quality varies significantly.

This showed me that having a strong scoring rubric will be important for maintaining consistency.

Cleaning and Data Wrangling Required

Several cleaning steps will be needed.

These include:

Standardizing prompt categories
Formatting response timestamps consistently
Measuring response length numerically
Organizing reviewer scores into structured columns

I also plan to create new features such as:

Average reviewer score
Score variance between reviewers

These features will make the final analysis easier and stronger.

Anticipated Challenges

One challenge is collecting enough reviewer feedback while keeping the project manageable.

Another challenge is reducing subjectivity in scoring. To address this, I plan to use a clear and simple rubric.

A final challenge is keeping the project focused and small enough to complete well.

# Mini Project 5: Working with Geospatial Data

## Project Overview

For this project, I analyzed Professor Wirfs-Brock’s location data from her Whitman interview visit in March 2022 using geospatial analysis and interactive mapping.

---

## Part A: Distance Analysis

I selected Memorial Building at Whitman College as my reference location.

Using the haversine formula, I calculated the distance between Memorial Building and every location point in the dataset.

The closest Professor Wirfs-Brock came to Memorial Building was approximately **0.0468 km** at **2022-03-11 20:10:01**.

The furthest point was approximately **1306.56 km** at **2022-03-12 21:34:38**, which likely represents travel outside Walla Walla.

---

## Part B: Movement Hypothesis

My hypothesis was that Professor Wirfs-Brock’s visit was mostly centered around Whitman College and nearby Walla Walla locations, while the largest distances occurred when she was traveling to or from the visit.

The interactive map shows that most location points are clustered around Walla Walla and the Whitman campus area, suggesting that most of her time was spent near campus for meetings, interviews, and visit-related activities.

The distance-over-time graph provides stronger evidence. At the beginning of the dataset, the distance from Memorial Building is very large, suggesting travel to Walla Walla. During most of the visit, the distance remains very low, meaning she stayed close to campus. At the end of the dataset, the distance increases sharply again, suggesting travel away after the visit.
This supports a clear **travel -> campus visit -> travel** pattern.
